Shutterstock at Scale

The infrastructure that powers discovery

  • 500M+ creative assets (Images, Video, Music, Sound FX)
  • 2M+ contributors worldwide with daily growth
  • 500K+ daily search sessions across 25+ languages
  • AWS-native stack: OpenSearch, S3, SageMaker, Lambda, Bedrock
📈

25 Years of IR Innovation

Evolution without deprecation

  • ~2000: Boolean & TF-IDF — the foundation
  • ~2004: BM25 & Inverted Index — precision at scale
  • ~2009: Learning to Rank — early ML integration
  • ~2018: BERT & Transformers — semantic understanding
  • ~2021: CLIP & Multimodal — cross-modal alignment
  • 2024+: Agentic Orchestration — intelligent composition

None are deprecated. All are tools in the agent's toolkit.

⚖️

Lexical (BM25) Search

Why term-based retrieval still wins

  • Exact Match Precision: SKU IDs, proper nouns — vectors hallucinate
  • Latency Profile: Sub-millisecond lookups; dense retrieval is 10-50x slower
  • Cold Start Reliability: Works day one for new assets without embeddings
  • Explainability: Every score is auditable for debugging and editorial trust
🎬

Multimodal Search

Users don't think in modalities — we bridge that gap

  • Image: CLIP embeddings + visual similarity for reference matching
  • Video: Frame-level CLIP + temporal signals for motion & mood
  • Music: Audio embeddings + BPM/mood taxonomy for audio discovery
  • Voice: ASR → intent → structured query for natural expression

All modalities unified in OpenSearch for the agent.

🔗

Hybrid Fusion Patterns

When 1 + 1 equals more than 2

  • RRF (Reciprocal Rank Fusion): Merge BM25 + vector rankings without score normalization. Best for general discovery.
  • Weighted Combo: α·BM25 + β·semantic with tuning per query class. Best for known-item + semantic mix.
  • Cascade Re-rank: BM25 retrieves, cross-encoder re-ranks top-K. Best for precision-critical editorial.

Agent selects fusion pattern at query time based on detected intent.

🤖

Agentic Search Patterns

Four behaviors that distinguish agentic IR

  • Reflection: Evaluate own output, detect gaps, re-query with refined parameters
  • Planning: Decompose complex queries into sub-goals before retrieval
  • Tool Use: BM25, dense retrieval, CLIP, knowledge graph — each as discrete tools
  • Multi-Agent: Specialized sub-agents per modality; coordinator fuses results
⚙️

Agentic Workflow

From intent to discovery in four steps

  • 01 Search Intent Detection: LLM classifies query intent (Discovery, Known-item, Similarity, Refine)
  • 02 Entity Extraction: Extract subject, style, mood, modality, license, era, brand
  • 03 LLM Query Strategy: Decide tools, fusion pattern, filters → execution plan
  • 04 OpenSearch via MCP: Execute BM25, k-NN, hybrid endpoints; fuse results
🛠️

MCP: Agent Tools for OpenSearch

Structured retrieval at the LLM layer

  • Tool Schema: opensearch_search(query, mode, filters, top_k)
  • Modes: bm25 | knn | hybrid
  • Reflection: Agent evaluates result quality; retries with refined params
  • Tool Use: Each mode is a discrete capability LLM can reason about
  • Planning: Filters + mode selection happen before execution
🎯

Key Takeaways

Role-specific insights from production agentic search

  • For Search Engineers: Expose retrieval tools as discrete MCP endpoints; instrument all tool calls
  • For AI Engineers: Use Reflection & Planning before retrieval; multi-agent fan-out for modalities
  • For Engineering Managers: Measure hybrid search ROI; invest in eval frameworks to catch non-determinism

The orchestra was always here. We finally have a conductor.
